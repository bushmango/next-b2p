Summary
- Lodash is used broadly as a general utility library.
- No app usage of Lodash template, unset, or omit was found.
- No app usage of Function() tied to Lodash templating was found.
- Current remaining Lodash advisories are therefore most concerning if:
  - the app starts using _.template with untrusted imports keys, or
  - the app uses _.unset / _.omit with untrusted array paths, or
  - a transitive dependency uses those APIs unsafely.

Recommendation
- Do not make a broad Lodash replacement as part of the current dependency cleanup.
- Next best options:
  1. Keep Lodash pinned and document accepted residual risk because risky APIs are not used in app code.
  2. If the registry has a real lodash 4.18.x package available and compatible, test upgrading direct lodash separately.
  3. Replace direct imports with the existing common/lib/lodash wrapper over time for consistency.
  4. Consider gradually replacing simple usages with native JS only when touching nearby code; avoid a sweeping refactor.
- Add a lint/search guard or code review rule: do not introduce _.template, _.unset, or _.omit with untrusted input.
